# napcat-plugin-dengche

群聊等车列表管理插件：等车、跑路、发车、等车列表；支持白名单启用与管理员强制增删。

同时提供“喊车（@全体组队）”功能：群内发送关键词后，机器人先发一段文字，再按身份与冷却决定是否 @全体成员。

本插件仅在群聊内响应命令。

## 功能与命令

说明：下面所有命令都需要加命令前缀（默认 `/`，可在配置里修改）。

### 群成员命令

- `/等车`
    - 将发送者加入等车列表（若已存在则刷新加入时间并移动到末尾）
    - 回复包含当前等车人数与列表

- `/跑路`
    - 将发送者从等车列表移除
    - 回复包含当前等车人数与列表

- `/等车列表`
    - 查看当前等车列表

- `/发车`
    - at 当前列表内所有成员，并发送：`来不及解释了，快上车！`
    - 发送后清空该群等车列表

- `/等车帮助`（或 `/等车 帮助`）
    - 查看“等车”相关指令说明
    - 即使该群处于关闭状态，也允许使用（便于新群快速上手）

### 管理员命令

- `/等车 开`
    - 开启当前群的等车功能

- `/等车 关`
    - 关闭当前群的等车功能
    - 关闭状态下，除 `/等车 开/关/帮助` 与 `/等车帮助` 外其他命令均不触发

- `/等车列表删除 1`（或 2/3/...）
    - 按序号强制移除等车列表中的成员
    - 成功会回复当前列表；序号不存在会提示失败

- `/等车列表添加 123456` 或 `/等车列表添加 @某人`
    - 将目标成员添加到列表末尾
    - 会校验对方是当前群成员；校验失败会提示“找不到该成员”

### 喊车功能

#### 触发方式（无需命令前缀）

在群配置中设置“触发关键词”后：当群成员发送的消息内容与该关键词完全一致时触发。

触发流程：

1. 先发送配置的“文字内容”（可为空；为空则不发送）
2. 再根据触发者身份与冷却规则决定是否发送一条“@全体成员”

#### 管理员命令（查看本群关键词）

- `/喊车关键词`
    - 仅群主/管理员可用
    - 展示本群喊车配置（是否开启、触发关键词、冷却、@全体策略、文字内容）

## 自动清理规则

- 每次准备读/写等车列表前，都会检查加入时间
- 超过配置的超时时长会自动从列表移除

默认超时时长为 6 小时。

## 配置项

配置可通过 NapCat WebUI 修改（插件配置面板）：

- `commandPrefix`：命令前缀，默认 `/`
- `waitTimeoutHours`：等车列表自动移除超时（小时），默认 `6`
- 群启用策略：按 `groupConfigs.<群号>.enabled` 为准（默认关闭）。
    - 在群内使用命令 `/等车 开` / `/等车 关` 可开启/关闭本群功能。
    - WebUI 的“群与功能”页面也可进行同样的开关与配置。

### 分群配置（WebUI 群管理页）

每个群可单独配置“喊车”模块（存储在 `groupConfigs.<groupId>.shoutCar`）：

- `enabled`：是否开启
- `cooldownSeconds`：触发冷却（秒）。仅用于限制“@全体成员”的发送频率；不影响前置文字内容
- `rules`：关键词规则列表（多条）
    - `keyword`：触发关键词（群内发送该文本，完全匹配触发）
    - `text`：触发后发送的文字内容（先发，可为空）
    - `atAllWhenAdmin`：群主/管理员触发时是否 @全体成员
    - `atAllWhenMember`：普通成员触发时是否 @全体成员

## 数据持久化

等车列表会持久化到插件数据目录（`ctx.dataPath`）：

- `wait_list.json`

按群分别保存。

喊车模块的 @全体冷却状态也会持久化到插件数据目录：

- `shout_car_state.json`

## 开发与构建

```bash
pnpm install

# 完整构建（自动构建 WebUI + 后端 + 复制资源）
pnpm run build

# 后端 watch（每次构建后自动部署/热重载，需 napcat-plugin-debug 配合）
pnpm run dev

# WebUI 前端开发服务器
pnpm run dev:webui
```

## Git 同步脚本（本地）

仓库内提供一个简单脚本用于在本地选择：推送到 GitHub 或从 GitHub 拉取。

```bash
# 交互菜单（推荐）
./scripts/git-sync.sh

# 推送到 GitHub（如检测到改动会提示是否先提交）
./scripts/git-sync.sh push

# 从 GitHub 拉取（仅允许 fast-forward，避免自动产生 merge commit）
./scripts/git-sync.sh pull

# 查看状态
./scripts/git-sync.sh status
```

默认：分支 `main`，远程名 `origin`。
